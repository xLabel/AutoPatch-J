## 上下文

AutoPatch-J 已经使用短 LLM 调用做意图识别和 pending-review 路由分类。异常会被吞掉并回落到安全默认值。源码读取工具会规范化仓库路径，并在请求路径不存在时尝试按文件名查找候选。

## 目标 / 非目标

**目标：**

- 让分类器 fallback 可诊断，同时不增加普通模式噪音。
- 防止同名源码路径歧义导致读错文件。
- 除分类器失败和歧义路径需要更清晰处理外，保持现有任务路由语义。

**非目标：**

- 替换短 LLM 分类器。
- 增加新的用户命令。
- 修改 function_call 工具名。

## 决策

- 为 route 和 intent 分类诊断引入轻量结果对象。
- 尽量保留现有 public 分类方法的 enum 返回值，并为 workflow 使用新增诊断方法。
- 缺失源码路径存在歧义时返回带候选路径的错误。
- 只有在应用 focus 约束后仍唯一匹配时，才自动纠正缺失路径。

## 风险 / 取舍

- debug 输出可能暴露内部分类失败细节，因此保持简短。
- 歧义路径错误可能要求 LLM 多调用一次工具，但可以避免基于错误文件生成补丁。
